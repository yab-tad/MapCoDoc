# pandas.CategoricalIndex.map

CategoricalIndex.map(mapper, na_action=None)[source](https://github.com/pandas-dev/pandas/blob/main/pandas/core/indexes/category.py#L431-L499)#(https://pandas.pydata.org/pandas-docs/version/2.3.3/reference/api/pandas.CategoricalIndex.map.html#pandas.CategoricalIndex.map)
    Map values using input an input mapping or function.

Maps the values (their categories, not the codes) of the index to new
categories. If the mapping correspondence is one-to-one the result is a
CategoricalIndex(https://pandas.pydata.org/pandas-docs/version/2.3.3/reference/api/pandas.CategoricalIndex.html#pandas.CategoricalIndex) which has the same order property as
the original, otherwise an Index(https://pandas.pydata.org/pandas-docs/version/2.3.3/reference/api/pandas.Index.html#pandas.Index) is returned.

If a dict or Series(https://pandas.pydata.org/pandas-docs/version/2.3.3/reference/api/pandas.Series.html#pandas.Series) is used any unmapped category is
mapped to NaN. Note that if this happens an Index(https://pandas.pydata.org/pandas-docs/version/2.3.3/reference/api/pandas.Index.html#pandas.Index)
will be returned.

Parameters:
    mapper : function, dict, or Series
    Mapping correspondence.

Returns:
    pandas.CategoricalIndex or pandas.Index
    Mapped index.

See also

Index.map(https://pandas.pydata.org/pandas-docs/version/2.3.3/reference/api/pandas.Index.map.html#pandas.Index.map)
    Apply a mapping correspondence on an Index(https://pandas.pydata.org/pandas-docs/version/2.3.3/reference/api/pandas.Index.html#pandas.Index).

    Apply a mapping correspondence on a Series(https://pandas.pydata.org/pandas-docs/version/2.3.3/reference/api/pandas.Series.html#pandas.Series).

    Apply more complex functions on a Series(https://pandas.pydata.org/pandas-docs/version/2.3.3/reference/api/pandas.Series.html#pandas.Series).

Examples

```
>>> idx = pd.CategoricalIndex(['a', 'b', 'c'])
>>> idx
CategoricalIndex(['a', 'b', 'c'], categories=['a', 'b', 'c'],
                  ordered=False, dtype='category')
>>> idx.map(lambda x: x.upper())
CategoricalIndex(['A', 'B', 'C'], categories=['A', 'B', 'C'],
                 ordered=False, dtype='category')
>>> idx.map({'a': 'first', 'b': 'second', 'c': 'third'})
CategoricalIndex(['first', 'second', 'third'], categories=['first',
                 'second', 'third'], ordered=False, dtype='category')

```

If the mapping is one-to-one the ordering of the categories is
preserved:

```
>>> idx = pd.CategoricalIndex(['a', 'b', 'c'], ordered=True)
>>> idx
CategoricalIndex(['a', 'b', 'c'], categories=['a', 'b', 'c'],
                 ordered=True, dtype='category')
>>> idx.map({'a': 3, 'b': 2, 'c': 1})
CategoricalIndex([3, 2, 1], categories=[3, 2, 1], ordered=True,
                 dtype='category')

```

If the mapping is not one-to-one an Index(https://pandas.pydata.org/pandas-docs/version/2.3.3/reference/api/pandas.Index.html#pandas.Index) is returned:

```
>>> idx.map({'a': 'first', 'b': 'second', 'c': 'first'})
Index(['first', 'second', 'first'], dtype='object')

```

If a dict is used, all unmapped categories are mapped to NaN and
the result is an Index(https://pandas.pydata.org/pandas-docs/version/2.3.3/reference/api/pandas.Index.html#pandas.Index):

```
>>> idx.map({'a': 'first', 'b': 'second'})
Index(['first', 'second', nan], dtype='object')

```
